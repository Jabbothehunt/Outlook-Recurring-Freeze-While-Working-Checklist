Outlook Recurring Freezes — Troubleshoot While It’s Working

Use this when Outlook is currently working but the issue is recurring. Check items as you go — progress saves in this browser. Drop-down + Reset
Goal: Identify the root cause (add-in, OST/profile corruption, shared mailbox caching, mailbox size, indexing, or connectivity).
Tip: If the user can work in OWA (web mail) while Outlook freezes, the problem is usually local (profile/OST/add-ins).
1) Add-ins Check (most common)
  • Outlook → File → Options → Add-ins.
  • Manage: COM Add-ins → Go → review list.
  • Disable anything not required (common: Adobe PDF, Teams/Zoom, CRM, antivirus mail scanning) and test.
  • Check Disabled Items (Outlook may have disabled a crashing add-in).
If Safe Mode works, it’s almost always add-ins: Win+R → outlook.exe /safe
2) Mailbox Size / Shared Mailboxes
  • Check mailbox folder size: Right-click mailbox → Data File Properties → Folder Size.
  • Note if mailbox is very large (example: 10–15GB+), or user has many folders / shared mailboxes.
  • Check shared mailbox caching: File → Account Settings → Account Settings → double-click mailbox → More Settings → Advanced.
  • Consider unchecking: Download shared folders (if many shared mailboxes cause freezes).
3) OST Size & Health (cache corruption)
This is one of the best “while it’s working” checks.
  • Win+R → %LOCALAPPDATA%\Microsoft\Outlook
  • Check the user’s .ost file size (very large OSTs can cause hangs).
  • If recurring freezes continue, plan to rebuild OST (close Outlook, rename OST to .old, reopen Outlook).
Rebuild OST (later, if needed): Close Outlook → %LOCALAPPDATA%\Microsoft\Outlook → rename mailbox.ost to mailbox.old → reopen Outlook
4) Indexing / Search Health
  • Outlook → click Search bar → Search Tools → Indexing Status.
  • If stuck or high items remaining for a long time, rebuild index: Control Panel → Indexing Options → Advanced → Rebuild.
  • Confirm Windows Search service is running (services.msc → Windows Search).
5) Connection Status / Reconnect Loops
  • Hold Ctrl and right-click the Outlook tray icon → Connection Status.
  • Look for repeated disconnect/reconnect patterns or many failing connections.
  • If using VPN/Zscaler, test briefly on/off (per policy) to see if it correlates with freezes.
6) Event Viewer (find the real offender)
  • Open Event Viewer → Windows Logs → Application.
  • Filter or scan for errors around the freeze time (sources: Outlook, Office, Application Error, Microsoft 365 Apps).
  • Capture faulting module/add-in name if present (very useful for permanent fix).
7) Updates / Repair (when recurring)
  • Outlook → File → Office Account → Update Options → Update Now.
  • If still recurring: run Office Quick Repair (Settings → Apps → Microsoft 365 → Modify → Quick Repair).
  • If still not fixed: consider Online Repair (last resort).
8) Notes / Ticket Documentation
Log what you found while it’s working (add-in list, OST size, shared mailbox count, errors, etc.).
Printing tip: in the print dialog, choose “Save as PDF” if you want to attach to the ticket.
